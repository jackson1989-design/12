## STEP 3: Enable GitHub Pages (1 minute)

1. Click the **"Settings"** tab at the top of your repository

2. In the left sidebar, scroll down and click **"Pages"**

3. Under "Source" section:
   - Click the dropdown that says "None"
   - Select **"main"** (or "master")
   - Leave folder as **"/ (root)"**
   - Click **"Save"**

4. Wait 1-2 minutes (GitHub is building your site)

5. **Refresh the page**

6. You'll see a green box that says:
   **"Your site is published at https://YOUR-USERNAME.github.io/latimore-digital-portfolio/"**

## ğŸ†˜ Troubleshooting

**"I don't see the Pages option in Settings"**
- Make sure repository is **Public**, not Private
- Try refreshing the page
- Make sure you're logged into your own account

**"My site shows a 404 error"**
- Make sure file is named `index.html` (not `latimore-portfolio.html`)
- Wait 2-3 minutes after enabling Pages
- Check that Pages source is set to "main" branch

**"I can't upload files"**
- Make sure you're on YOUR repository page (not someone else's)
- Check you're logged in
- Try a different browser
